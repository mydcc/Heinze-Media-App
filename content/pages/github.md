## Meine Lieblings-GitHub-Projekte

Willkommen zur Sammlung meiner bevorzugten GitHub-Projekte. Hier findest du kurz beschriebene Projekte mit Link, Sprache, Tags und einer kurzen Begründung, warum ich sie empfehle.

<!-- Hinweis: Projektdateien werden aus `content/projects/` geladen. Jede Datei hat Frontmatter: title, repo, url, description, tags, stars, language, featured. -->

### So verwalte ich die Liste (Mini‑CMS‑freundlich)

- Einzeldateien pro Projekt: `content/projects/<slug>.md` — ideal für Git-basiertes CMS (z. B. Netlify CMS, TinaCMS).
- JSON-Alternative: `content/data/github-projects.json` für GUIs oder externe Editoren.
- Automatisches Sync-Skript: `scripts/sync_github_projects.js` (GitHub API) erzeugt/aktualisiert Projekt‑Markdowns.

### Beispiel-Auswahl

- **Project One** — Minimaler CLI-Task-Runner. [Repo](https://github.com/example/project-one)
- **Project Two** — Nützliche UI-Komponenten-Bibliothek. [Repo](https://github.com/example/project-two)
- **Project Three** — Datenvisualisierung mit Fokus auf Performanz. [Repo](https://github.com/example/project-three)

### Mitmachen

Wenn du ein Projekt vorschlagen willst, öffne bitte eine Issue oder sende mir einen Pull Request mit einer neuen Datei in `content/projects/`.
